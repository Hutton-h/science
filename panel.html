为什么浏览器提示"不安全"？

🔒

面板使用 busybox httpd 提供 HTTP 服务，没有内置 TLS/SSL 支持，浏览器会标记为"不安全"。这不影响代理节点本身的安全性——所有代理协议（VLESS、VMess、Hysteria2 等）都有独立的加密层。

面板仅用于查看节点信息和订阅链接，不传输敏感数据。

方案一：Cloudflare 代理（推荐）

最简单的方式：在 Cloudflare DNS 中将订阅域名设为🟠橙色云朵（代理模式），Cloudflare 会自动提供 HTTPS。浏览器通过 HTTPS 访问 Cloudflare，Cloudflare 通过 HTTP 回源到你的服务器。
前提：订阅面板地址用的是域名（如 http://你的域名:端口/token/index.html），且域名已解析到服务器 IP。

方案二：Nginx/Caddy 反向代理

在服务器上安装 Nginx 或 Caddy，配置反向代理 + Let's Encrypt 自动 HTTPS 证书：
# Caddy 示例（最简单） sudo apt install -y caddy echo "你的域名 { reverse_proxy localhost:订阅端口 }" > /etc/caddy/Caddyfile sudo systemctl restart caddy

方案三：Cloudflare Tunnel

使用 cloudflared tunnel 创建安全隧道，无需开放端口，自动获得 HTTPS。Argo 固定隧道本身就是这个原理。
